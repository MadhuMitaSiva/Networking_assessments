4. 
Class classification is based on the first octet
Class A -> 0xxxxxxx -> subnet 255.0.0.0 ; N=8 bits H=24 bits 
Class B -> 10xxxxxx -> subnet 255.255.0.0 ; N=16 bits H=16 bits 
Class C -> 110xxxxx -> subnet 255.255.255.0 ; N=24 bits H=8 bits 
a) 10.1.1.1 -> Binary of the 1st octet 10 => 00001010 -> Class A -> 10.0.0.0 - 10.255.255.255
b) 172.16.5.10 -> Binary of the 1st octet 172=> 10101100 -> Class B -> 172.0.0.0 - 172.255.255.255
c) 192.168.1.5 -> Binary of the 1st octet 192 => 11000000 -> Class C -> 192.0.0.0 - 192.255.255.255
Since all lie with the range they are private

The private ip is converted into public ip by the nat to connect to the internet
